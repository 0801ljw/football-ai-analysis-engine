# 三主题工作台视觉改版设计

**状态：用户已确认视觉方向，待实施**

## 目标
为“世界杯 AI 推演内容引擎”提供同一信息架构下的三套可切换主题，默认使用 Claude 风格浅色研究工作台。

## 主题

### 1. Claude 研究工作台（默认）
- 用途：首次使用、赛前报告阅读、长时间研究。
- 语言：暖灰画布、白色表面、克制边界、大留白、低视觉噪音。
- 色彩：暖灰与石墨黑为基底，单一深橙作为主操作色，绿色仅表达真实成功状态。
- 字体：中文优先 `PingFang SC` / `Noto Sans CJK SC`，数据用 `JetBrains Mono` / 系统 Mono。

### 2. 美加墨联办指挥台（深色）
- 用途：运行队列、数据质量、报告生成和引擎状态。
- 语言：深色数据指挥台，线框网格，等宽数据标签。
- 色彩：午夜蓝基底；Canada red、Mexico green、USA navy 仅用于顶栏与对应区域；奖杯金用于主操作和当前选择；绿色仅表达 ready/success。
- 声明：是“美加墨联办灵感”，不使用 FIFA 官方 logo、官方字体或受保护图形资产，也不宣称官方关系。

### 3. 深蓝科技
- 用途：保留已有报告视觉系统的延续性，适合展示型输出。
- 语言：深蓝基底，青蓝信息高亮，低饱和紫青用于次级状态。
- 约束：不改变功能或文案含义，仅替换 tokens 与 component chrome。

## 交互与持久化
- 首页顶栏提供显式主题切换，不改变 URL、表单字段、API 请求或 run artifact。
- 用 `localStorage` 保存主题键 `wc-theme`；初次访问默认 `claude`。
- 三套主题使用 CSS variables，通过 `body[data-theme]` 或等价根节点选择器切换，禁止维护三份 HTML。
- 所有主题均需支持移动端、focus state、表单错误/空/加载状态；`prefers-reduced-motion` 下无非必要动效。

## 不变项
- `/api/*`、认证、run 创建/取消/重试、导出、数据可信度逻辑均不改。
- 不新增第三方前端依赖，不引入远程 Google Fonts；优先系统字体栈，避免网络失败影响首屏。
- 合规文案与输出边界保持不变。

## 验收
1. 主题切换只改视觉，现有页面交互仍可用。
2. 刷新后主题保持。
3. 默认无 localStorage 时显示 Claude 主题。
4. 三主题在桌面和窄屏下无水平溢出。
5. `pytest -q`、`pip check`、`node --check app/static/app.js`（若存在）通过。
6. 浏览器实测：切换主题、创建 dry-run、打开/导出历史 run。
