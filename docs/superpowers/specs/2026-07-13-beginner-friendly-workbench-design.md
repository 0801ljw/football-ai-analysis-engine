# PitchMind 新手友好融合首页设计

**日期：** 2026-07-13

**状态：** 用户已批准执行

**目标：** 以 C「双栏工作台」为主体，融合 A「三步新手引导」和 B「关键状态总览」，让首次使用者无需理解技术术语即可完成第一份报告。

## 1. 用户问题

当前首页把普通工作流、Token、Skill、Runtime doctor、数据源、dry-run、Admin、原始 API 等能力铺在同一页面，层级混乱。新手无法判断：先做什么、哪些字段必须填、什么时候完成、失败后怎么办。

## 2. 核心信息架构

### 顶部导航
- 品牌：PitchMind / 足球赛事 AI 推演引擎。
- 主导航：开始、历史报告。
- 次级入口：设置与高级工具。
- 不再在首页暴露主题切换、Skill bridge、API 路径等技术信息。

### 轻量状态总览（融合 B）
只展示三项与普通用户相关的状态：
1. 数据源：已就绪 / 需要配置；
2. 最近报告：完成数量；
3. 当前任务：无任务 / 排队 / 获取数据 / 生成报告。

### 新手任务条（融合 A）
`1 选择比赛 → 2 生成报告 → 3 查看与导出`
- 首次进入持续显示；完成一次报告后弱化。
- 当前步骤高亮。
- 提供“一键体验 Demo”。

### 主工作区（采用 C）
桌面宽屏为双栏：
- 左侧 5/12：新建报告；
- 右侧 7/12：最近报告。

窄屏降为单栏，顺序为状态 → 引导 → 新建报告 → 最近报告。

## 3. 左栏：新建报告

普通用户只看到：
- 官方场次编号；
- “检查可用场次”；
- 可用场次选择结果；
- 报告风格（深色看板 / 紫青科技 / 深蓝科技）；
- 数据模式：
  - 体验模式：使用示例数据；
  - 真实模式：获取公开赛事数据；
- 唯一主操作：`开始生成报告`。

报告标题自动生成，超时参数保持系统默认，不暴露给普通用户。体验 Demo 可预填示例编号并以体验模式直接运行。

## 4. 右栏：最近报告

- 默认加载最近记录；
- 每条显示比赛编号、标题、时间/状态、可信度；
- 主要动作：查看报告、导出、失败时重试；
- 运行中显示明确进度；
- 无记录时显示“从左侧生成第一份报告”的空状态。

## 5. 渐进式披露

以下内容保留功能但移入“设置与高级工具”折叠区：
- 账号 Token、桌面数据源 API Key；
- Runtime doctor、Setup guide、Skill 状态；
- 原始 JSON、dry-run 命令、路径、timeout；
- 队列、失败诊断、Admin 管理、API 预览；
- 主题切换。

普通首页文案不出现 `X-API-Token`、`run_id`、`skill bridge`、`runs/` 等术语。

## 6. 视觉规范

- 暖灰背景、深海军蓝导航、单一青绿色强调色；不使用紫蓝 AI 渐变作为主视觉。
- 内容最大宽度约 1280px；Windows 1366×768 首屏应能看到状态、任务条及双栏主操作。
- 卡片不全部使用同样阴影；以间距、背景层级和细边界区分。
- 主按钮明显；次级动作降权；所有交互有 hover、active、focus-visible。
- 字体使用系统字体，数字启用 tabular figures；不增加外部字体依赖。

## 7. 行为与兼容

- 保留现有 API、DOM 交互和功能语义，重组而非重写后端。
- 继续支持 desktop mode 和本机 API Key 管理。
- 体验模式默认开启；真实模式必须有清晰说明。
- 合规提示压缩成主工作区底部一句，完整说明放高级区。
- 无新增第三方依赖，不切换 npm 工具链。

## 8. 验收标准

1. 首页首屏只呈现普通用户需要的状态、三步引导、创建入口和最近报告；
2. 三步流程 DOM 与既有 API 行为保持可用；
3. 高级能力全部可访问但默认不干扰主流程；
4. 桌面 1366×768 无横向溢出，主操作首屏可见；
5. 390px 窄屏无页面级横向溢出；
6. 键盘焦点可见，核心控件有可访问标签；
7. 原有测试全绿，并新增首页信息架构回归测试；
8. 浏览器真实走通 Demo：选择/检查 → 生成 → 进度/结果；
9. 不改变后端 API、数据模型、发布 workflow 或依赖。
